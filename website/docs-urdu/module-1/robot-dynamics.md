# روبوٹ ڈائنامکس

## روبوٹ ڈائنامکس کو سمجھنا

روبوٹ ڈائنامکس طاقتیں اور ٹورک کا مطالعہ ہے جو روبوٹک نظام میں حرکت کو پیدا کرتے ہیں۔ ہیومنوڈ روبوٹس کے لیے، ڈائنامکس کو سمجھنا مستحکم لوکوموشن، درست مینیولیشن، اور ماحول کے ساتھ محفوظ تعامل کے لیے اہم ہے۔ سادہ نکاتی ماسز کے برعکس، ہیومنوڈ روبوٹس پیچیدہ متعدد جسم کے نظام ہیں جن کے کئی ڈگری آف فریڈم ہیں جن کو ایک ہی وقت میں کنٹرول کیا جانا چاہیے۔

### ڈائنامکس کیوں اہم ہیں

روبوٹ ڈائنامکس کنٹرول کرتے ہیں:

- کیسے طاقتیں روبوٹ کی حرکت کو متاثر کرتی ہیں
- حرکت کے دوران توانائی کی کھپت
- استحکام اور توازن کی برقرار رکھنا
- بیرونی اشیاء کے ساتھ تعامل
- کام کے دوران حفاظت

### کلیدی تصورات

#### ڈگریز آف فریڈم (DOF)
میکینیکل نظام کی تشکیل کو بیان کرنے والے آزاد پیرامیٹر کی تعداد۔ ہیومنوڈ روبوٹس کو انسان نما حرکت اور مہارت حاصل کرنے کے لیے عام طور پر 30+ DOF ہوتے ہیں۔

#### تشکیل کا علاقہ
روبوٹ کی تمام ممکنہ تشکیلات کا علاقہ، جوڈوں کے زاویوں اور مقامات کے ذریعے بیان کیا گیا ہے۔

#### حالت کا علاقہ
روبوٹ کی تمام ممکنہ حالت کا علاقہ، بشمول مقامات، رفتار، اور ممکنہ طور پر تیزی۔

## ریاضی کی بنیادیں

### نیوٹن-ایولر کی تشکیل

نیوٹن-ایولر کے مساوات جڑیل جسموں کی حرکت کی وضاحت کرتے ہیں:

**ترجمہ:**
```
F = ma
```

**گردش:**
```
τ = Iα + ω × (Iω)
```

جہاں:
- F: قوت ویکٹر
- τ: ٹورک ویکٹر
- m: ماس
- a: لکیری تیزی
- I: انیشیا ٹینسر
- α: زاویہ وار تیزی
- ω: زاویہ وار رفتار

### لاگرانجین کی تشکیل

پیچیدہ متعدد جسم کے نظام کے لیے لاگرانجین کا نقطہ نظر اکثر ترجیح دیا جاتا ہے:

```
L = T - V
```

جہاں:
- L: لاگرانجین
- T: کائینیٹک توانائی
- V: ممکنہ توانائی

حرکت کے مساوات مندرجہ ذیل ہیں:

```
d/dt(∂L/∂q̇) - ∂L/∂q = Q
```

جہاں:
- q: جنرلائزڈ کوآرڈینیٹس
- Q: جنرلائزڈ فورسز

## فارورڈ اور ان ورس ڈائنامکس

### فارورڈ ڈائنامکس

جوڈوں کے ٹورک اور موجودہ حالت دی گئی، جوڈوں کی تیزی کا حساب:

```
τ = H(q)q̈ + C(q,q̇)q̇ + g(q) + J^T F_ext
```

جہاں:
- H(q): ماس میٹرکس
- C(q,q̇): کوریولس اور سنٹریفوگل قوتیں
- g(q): گریویٹیشنل قوتیں
- J: جیکوبین میٹرکس
- F_ext: بیرونی قوتیں

تیزی کے لیے حل:

```
q̈ = H⁻¹(τ - Cq̇ - g - J^T F_ext)
```

### ان ورس ڈائنامکس

مطلوبہ حرکت دی گئی، ضروری جوڈوں کے ٹورک کا حساب:

```
τ = H(q)q̈_des + C(q,q̇)q̇ + g(q) + J^T F_ext
```

## ہیومنوڈ روبوٹس کے لیے متعدد جسم کا ڈائنامکس

### کنیمیٹک چینز

ہیومنوڈ روبوٹس متعدد کنیمیٹک چینز پر مشتمل ہیں:

- **لیگ چینز**: لوکوموشن اور توازن کے لیے
- **آرم چینز**: مینیولیشن اور تعامل کے لیے
- **ٹرونک**: منسلک اور مستحکم کرنے والا عنصر
- **ہیڈ**: ادراک اور مواصلت کے لیے

### مرکزِ کثافت (CoM)

مرکزِ کثافت توازن اور استحکام کے لیے اہم ہے:

```
CoM = Σ(m_i * r_i) / Σ(m_i)
```

جہاں:
- m_i: i لک کی ماس
- r_i: i لک کے مرکزِ کثافت کا مقام

### زیرو مومینٹ پوائنٹ (ZMP)

ZMP دو پائی استحکام کے لیے ایک اہم تصور ہے:

```
ZMP_x = (Σ(m_i * (ẍ_i * h - z_i * ẍ_i)) / Σ(m_i * g + m_i * z̈_i))
ZMP_y = (Σ(m_i * (ÿ_i * h - z_i * ÿ_i)) / Σ(m_i * g + m_i * z̈_i))
```

جہاں:
- h: CoM کی زمین کے اوپر اونچائی
- g: گریویٹیشنل تیزی

## کنٹرول کی حکمت عملی

### کمپیوٹڈ ٹورک کنٹرول

روبوٹ ڈائنامکس کی تلافی کر کے مطلوبہ رویہ حاصل کرنا:

```python
def computed_torque_control(q, qdot, q_des, qdot_des, qddot_des):
    # مطلوبہ ٹورک کا حساب
    q_error = q_des - q
    qdot_error = qdot_des - qdot

    # فیڈ فارورڈ + فیڈ بیک کنٹرول
    tau_ff = H(q) @ qddot_des + C(q, qdot) @ qdot_des + g(q)
    tau_fb = Kp @ q_error + Kd @ qdot_error

    return tau_ff + tau_fb
```

### آپریشنل سپیس کنٹرول

جوڈ سپیس کے قیود کو دیکھ بھال کرتے ہوئے ٹاسک سپیس ویریبلز کو کنٹرول کرتا ہے:

```python
def operational_space_control(x_des, xdot_des, xddot_des, J, H, C, g):
    # ٹاسک سپیس ماس میٹرکس
    Lambda = inv(J @ inv(H) @ J.T)

    # ٹاسک سپیس ڈائنامکس
    x_error = x_des - x
    xdot_error = xdot_des - xdot

    # مطلوبہ ٹاسک سپیس فورس
    f_des = Lambda @ (xddot_des + Kp @ x_error + Kd @ xdot_error)

    # جوڈ ٹورک
    tau = J.T @ f_des + gravity_compensation(H, C, g)

    return tau
```

### امپیڈنس کنٹرول

رابطے کے نقاط پر مطلوبہ ڈائنامک رویہ پیدا کرتا ہے:

```python
def impedance_control(x_des, xdot_des, xddot_des, x, xdot,
                     M_d, D_d, K_d):
    # امپیڈنس خطا
    x_error = x_des - x
    xdot_error = xdot_des - xdot

    # مطلوبہ امپیڈنس ڈائنامکس
    f_imp = M_d @ (xddot_des - xddot) + \
            D_d @ xdot_error + K_d @ x_error

    return f_imp
```

## توازن اور استحکام

### سٹیٹک بمقابلہ ڈائنامک توازن

**سٹیٹک توازن**: CoM ہمیشہ سپورٹ پولی گون کے اندر رہتا ہے
**ڈائنامک توازن**: سسٹم فعال کنٹرول اور مومینٹم مینجمنٹ کے ذریعے توازن برقرار رکھتا ہے

### کیپچر پوائنٹ

کیپچر پوائنٹ اس بات کی نشاندہی کرتا ہے کہ CoM موجودہ مومینٹم کو دیکھتے ہوئے کہاں آرام کرے گا:

```
کیپچر پوائنٹ = CoM + (CoM_velocity / sqrt(g/height))
```

### توازن کنٹرول کی حکمت عملی

1. **انکل کی حکمت عملی**: چھوٹی رکاوٹوں کو انکل ٹورک کے ذریعے درست کیا جاتا ہے
2. **ہپ کی حکمت عملی**: بڑی رکاوٹوں کو ہپ کی حرکت کے ذریعے سنبھالا جاتا ہے
3. **اسٹیپنگ کی حکمت عملی**: فوٹ پلیسمنٹ کے ذریعے بازیافت
4. **مومینٹم کنٹرول**: توازن کے لیے زاویہ وار مومینٹم کا استعمال

## شبیہ سازی کی مثال

یہاں ڈائنامکس شبیہ سازی کی ایک سادہ مثال ہے:

```python
import numpy as np
from scipy.integrate import odeint

class SimpleHumanoid:
    def __init__(self):
        # سادہ ماڈل: الٹا پینڈولم
        self.mass = 75.0  # kg
        self.height = 1.0  # m
        self.gravity = 9.81  # m/s^2

    def dynamics(self, state, t, control_torque):
        # حالت: [زاویہ, زاویہ وار رفتار]
        theta, theta_dot = state

        # الٹا پینڈولم کے لیے حرکت کا مساوات
        theta_ddot = (self.gravity / self.height) * np.sin(theta) + \
                     control_torque / (self.mass * self.height**2)

        return [theta_dot, theta_ddot]

    def simulate_step(self, state, dt, control_torque):
        # سادہ اولر انٹیگریشن
        theta, theta_dot = state
        theta_ddot = (self.gravity / self.height) * np.sin(theta) + \
                     control_torque / (self.mass * self.height**2)

        new_theta_dot = theta_dot + theta_ddot * dt
        new_theta = theta + new_theta_dot * dt

        return [new_theta, new_theta_dot]

# مثال کا استعمال
robot = SimpleHumanoid()
state = [0.1, 0.0]  # ابتدائی زاویہ اور رفتار
dt = 0.01  # 100 Hz کنٹرول

for i in range(1000):
    # سادہ توازن کنٹرول
    balance_torque = -10 * state[0] - 5 * state[1]  # PD کنٹرول
    state = robot.simulate_step(state, dt, balance_torque)

    if i % 100 == 0:
        print(f"وقت: {i*dt:.2f}s, زاویہ: {state[0]:.3f} rad")
```
